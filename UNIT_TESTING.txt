Unit Testing Documentation

This document provides an overview of the unit tests for the PasswordPal Frontend (UI Components), how to run them, and the expected output for each test case.

Running Tests

To run the unit tests, execute the following command in the terminal within the PasswordPal_Frontend directory:

npm test

This will trigger the vitest runner defined in package.json.

Notes

- Scope: These tests cover UI components only (React components in the src folder).
- Not Covered: Backend Rust logic, Tauri API calls, actual encryption/decryption, database operations.
- Testing Framework: Vitest + React Testing Library + jsdom
- No Build Tools Required: All tests run in Node.js environment without Visual Studio Build Tools.
- Mock Data: Tests use mock functions (vi.fn()) to simulate callbacks and user interactions.
- DOM Simulation: jsdom provides a browser-like environment for testing React components.

Troubleshooting

If tests fail:
1. Check that all dependencies are installed: npm install
2. Verify Node.js version is compatible (v16 or higher recommended)
3. Review error messages for specific test failures
4. Ensure no syntax errors in component files
5. Check that test files are in src/tests/ directory

For detailed test failure information, run:
npm test -- --reporter=verbose
