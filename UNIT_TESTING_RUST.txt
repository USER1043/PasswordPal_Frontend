Unit Testing Documentation – Rust component

This document provides an overview of the unit tests for the Password Pal client side’s backend (rust components), how to run them, and the expected output for each test case.

Running Tests:
To run the unit tests, execute the following command in the terminal within the PasswordPal_Frontend directory:
```bash
Cargo test
```
This will trigger the cargo test runner.

Test Suites Breakdown:

1. Auth Tests

Test: test_change_password_invalid_base64_blob
Expected: The function should return an error when the provided password blob is not a valid base64 encoded string.

Test: test_change_password_blob_too_short
Expected: The function should return an error when the password blob length is less than the expected minimum.

Test: test_change_password_invalid_salt
Expected: The function should return an error when the salt is not a valid base64 string or is of incorrect length.

Test: test_register_vault_success
Expected: The vault is successfully initialized and the encrypted master key is stored.

Test: test_login_vault_failure_wrong_password
Expected: Login fails with an unauthorized or incorrect password error.

Test: test_change_password_invalid_old_password
Expected: Password change fails if the provided old password does not match the stored one.

Test: test_login_vault_success
Expected: User successfully authenticates and the vault is unlocked.

Test: test_change_password_empty_passwords
Expected: Password change fails if the provided old or new password is empty.

Test: test_change_password_success
Expected: Password is changed successfully and the vault is updated.

Test: test_change_password_with_special_characters
Expected: Password is changed successfully even when containing special characters.

Test: test_change_password_multiple_times
Expected: Password can be changed multiple times successfully.

2. Entry Tests

Test: test_decrypt_blob_too_short
Expected: The function should return an error when the encrypted blob is too short to be valid.

Test: test_decrypt_invalid_base64
Expected: The function should return an error when the encrypted blob is not a valid base64 encoded string.

Test: test_decrypt_fail_when_locked
Expected: The function should return an error when trying to decrypt an entry in a locked vault.

Test: test_decrypt_fail_tampered_data
Expected: The function should return an error when the encrypted data has been tampered with (authentication check fails).

Test: test_encrypt_entry_with_empty_fields
Expected: The function should successfully encrypt an entry even if some fields (like URL or notes) are empty.

Test: test_encrypt_entry_with_special_characters
Expected: The function should successfully encrypt an entry containing special characters in the name, username, or password.

Test: test_decrypt_with_wrong_key
Expected: The function should return an error when attempting to decrypt with an incorrect key (simulating wrong master password).

Test: test_encrypt_entry_with_unicode_characters
Expected: The function should successfully encrypt an entry containing Unicode characters (e.g., emojis, non-ASCII).

Test: test_encrypt_fail_when_locked
Expected: The function should return an error when trying to encrypt an entry in a locked vault.

Test: test_encrypt_decrypt_flow
Expected: A successful round trip of encrypting an entry and then decrypting it should yield the original data.

Test: test_encrypt_multiple_entries_sequentially
Expected: Multiple entries should be encrypted successfully one after another without affecting each other.

Test: test_encrypt_entry_with_very_long_url
Expected: The function should handle and encrypt entries with very long URLs.

Test: test_encrypt_large_entry
Expected: The function should handle and encrypt entries with large amounts of data in the fields.

Expected: The vault state should remain unchanged (locked) after a failed unlock attempt.

Expected: The function should return an error when the provided password blob has an incorrect length.
